# Week 6: Assembly Integration Project

**Theme**: Comprehensive Assembly Programming
**Duration**: 5 days (50 minutes each)
**Learning Objectives**:
- Integrate all assembly concepts
- Design complex multi-feature programs
- Apply software engineering principles
- Demonstrate assembly mastery

---

## Week 6, Day 1: Project Planning & Design

**Topic**: Algorithm Design and Project Specification

### Lesson Overview

**Project Introduction**:
Students design and implement a comprehensive assembly program incorporating:
- Data movement (registers, memory, stack)
- Arithmetic operations
- Logic operations
- Control flow (loops, conditionals)
- User interaction (I/O)

### Project Options

**Option 1: Advanced Calculator**
- Multiple operations
- Memory functions
- History using stack
- Menu system with loops

**Option 2: Data Analysis Tool**
- Read multiple values
- Calculate statistics (min, max, average, sum)
- Sort or filter data
- Report results

**Option 3: Simple Game**
- Number guessing with lives
- Score tracking
- Multiple rounds
- Win/loss conditions

**Option 4: Text Processor**
- Read ASCII characters
- Process/transform
- Store results
- Output formatted

**Option 5: Student Choice** (requires approval)
- Must use all concept areas
- Clear specification
- Feasible in time frame

### Requirements Checklist

**Must Include**:
- [ ] Memory storage (STM/LDM)
- [ ] Stack usage (PUSH/POP)
- [ ] Arithmetic (ADD/SUB)
- [ ] Logic operations (AND/OR/XOR/NOT)
- [ ] Comparison (CMP)
- [ ] Conditional jumps (JZ/JNZ/JC/JNC)
- [ ] At least one loop
- [ ] User input (IN)
- [ ] User output (OUT)
- [ ] Comments and documentation

### Day 1 Activities

**Design Phase** (40 minutes):
1. Choose project (10 min)
2. Write specification (10 min)
3. Create algorithm/flowchart (10 min)
4. Plan data structures (10 min)

**Peer Review** (10 min):
- Partner feedback on design
- Feasibility check
- Suggestions

**Homework**: Refine design, start coding

## Project Rubric (100 points)

### Functionality (40 points)
- **Complete (40)**: All features work perfectly
- **Mostly Complete (32)**: Minor issues, all features present
- **Partially Complete (24)**: Some features work
- **Incomplete (16)**: Many features missing/broken
- **Minimal (8)**: Basic functionality only

### Required Components (30 points)
- Memory usage (3)
- Stack usage (3)
- Arithmetic operations (3)
- Logic operations (3)
- Comparisons (3)
- Conditional jumps (3)
- Loops (3)
- User I/O (3)
- Comments (3)
- Documentation (3)

### Code Quality (15 points)
- **Excellent (15)**: Clear, organized, efficient
- **Good (12)**: Well-structured, readable
- **Adequate (9)**: Functional but messy
- **Poor (6)**: Disorganized, hard to follow
- **Minimal (3)**: Barely functional

### Design & Creativity (10 points)
- **Outstanding (10)**: Creative, well-designed
- **Good (8)**: Solid design choices
- **Adequate (6)**: Basic design
- **Minimal (4)**: Little design consideration

### Documentation (5 points)
- **Comprehensive (5)**: Complete docs, user guide
- **Good (4)**: Good comments, some docs
- **Adequate (3)**: Basic comments
- **Minimal (2)**: Few comments
- **None (0)**: No documentation

### Bonus Points (up to +20)
- Exceptional complexity (+5)
- Outstanding user experience (+5)
- Creative features (+5)
- Helping others (+3)
- Code optimization (+2)

## Teacher Notes

### Time Management:
- Days 1-2: Must have working core
- Day 3: Feature freeze
- Day 4: Testing only
- Day 5: Presentations ready

### Common Project Issues:
- **Scope too large**: Help students cut features
- **Stack underflow**: Check PUSH/POP balance
- **Infinite loops**: Debug with Step mode
- **Lost in complexity**: Refocus on requirements

### Success Indicators:
- All required concepts present
- Working demonstration
- Clear documentation
- Student can explain code

### Grading Efficiency:
- Live demos count for functionality
- Code review during work time
- Documentation graded separately
- Rubric scoring during presentations
